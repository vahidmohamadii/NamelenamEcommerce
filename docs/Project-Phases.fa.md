# فازبندی پروژه فروشگاه اینترنتی NaderEcommerce

منبع اصلی تصمیم‌گیری: `C:\Users\Lenovo\Downloads\E-Commerce-Web-Application-Proposal.md`

هر وقت مسیر پروژه مبهم شد، اول به همان پروپوزال رجوع کن و بعد این فازبندی را با وضعیت واقعی کد هماهنگ کن.

## خلاصه پروژه

هدف ساخت یک پلتفرم فروشگاهی چنددسته‌ای با معماری تمیز، بک‌اند ASP.NET Core 8 Web API، فرانت‌اند Blazor Web App روی .NET 8، دیتابیس SQL Server، احراز هویت JWT/Refresh Token، پنل مدیریت کامل، پرداخت آنلاین، سبد خرید، علاقه‌مندی‌ها، CMS و پشتیبانی QR Code برای محصولات است.

## فاز 0: آماده‌سازی و اسکلت پروژه

هدف: ساخت پایه‌ای که بقیه پروژه روی آن بدون شلوغی جلو برود.

کارها:
- ایجاد Solution و پروژه‌های معماری Clean Architecture:
  - `Domain`
  - `Application`
  - `Infrastructure`
  - `WebApi`
  - `BlazorWeb`
- تنظیم Dependency Injection بین لایه‌ها.
- اضافه کردن EF Core، SQL Server، Swagger، Serilog، AutoMapper و FluentValidation.
- تعریف تنظیمات محیطی برای Development و Production.
- آماده‌سازی ساختار Git، فایل README و راهنمای اجرای اولیه.

خروجی قابل تحویل:
- پروژه قابل Build
- Swagger فعال
- اتصال اولیه به SQL Server
- ساختار معماری مشخص

معیار اتمام:
- Solution بدون خطا Build شود.
- API با Swagger بالا بیاید.
- یک Health Check یا Endpoint ساده پاسخ بدهد.

## فاز 1: دامنه، دیتابیس و احراز هویت

هدف: ساخت ستون فقرات سیستم شامل کاربران، نقش‌ها، امنیت و مدل‌های اصلی دیتابیس.

کارها:
- طراحی Entityهای اصلی:
  - User, Role, UserRole
  - Product, ProductImage, Category, ProductCategory
  - Order, OrderItem, Payment
  - Review, Wishlist, Coupon
  - QRLink, WebsiteSettings, Slider, Page
- پیاده‌سازی EF Core DbContext و Migrationها.
- پیاده‌سازی Repository Pattern یا الگوی موجود در پروژه.
- پیاده‌سازی Register/Login.
- پیاده‌سازی JWT Authentication و Refresh Token.
- پیاده‌سازی Role Based Authorization برای Customer و Admin.
- اضافه کردن Password Hashing و Validationها.

خروجی قابل تحویل:
- دیتابیس اولیه
- سیستم ثبت‌نام و ورود
- نقش‌های Customer و Administrator
- Endpointهای پایه User/Auth

معیار اتمام:
- کاربر بتواند ثبت‌نام و لاگین کند.
- توکن و Refresh Token درست کار کند.
- Endpointهای Admin فقط با نقش Admin قابل دسترسی باشند.

## فاز 5: امنیت، کارایی و SEO

هدف: آماده‌سازی پروژه برای استفاده جدی‌تر و کاهش ریسک‌های فنی.

کارها:
- اعمال HTTPS.
- سخت‌سازی JWT و Refresh Token.
- Rate Limiting.
- محافظت در برابر SQL Injection، XSS و CSRF.
- Logging ساختاریافته با Serilog.
- Pagination در همه لیست‌های سنگین.
- Async APIها.
- Caching برای داده‌های مناسب.
- Image Optimization.
- Database Indexes.
- Response Compression.
- SEO:
  - URLهای خوانا
  - Meta Tags
  - OpenGraph
  - XML Sitemap
  - robots.txt
  - Canonical URLs

خروجی قابل تحویل:
- نسخه امن‌تر و سریع‌تر
- SEO پایه
- لاگ‌های قابل پیگیری

معیار اتمام:
- صفحات اصلی سریع بارگذاری شوند.
- لیست‌ها بدون فشار غیرضروری روی دیتابیس کار کنند.
- اسکن اولیه امنیتی و تست‌های دستی مسیرهای مهم بدون ایراد جدی باشند.

وضعیت اجرا:
- HTTPS redirection، HSTS در Production و پشتیبانی از Forwarded Headers برای اجرا پشت Nginx اضافه شد.
- JWT سخت‌تر شد: اعتبارسنجی issuer/audience/lifetime/signature، الگوریتم مجاز، عدم ذخیره token و RequireHttpsMetadata در Production.
- Refresh Tokenها همچنان hash و rotate می‌شوند و در تغییر رمز/غیرفعال‌سازی کاربر revoke می‌شوند.
- Rate Limiting عمومی برای API و محدودیت سخت‌تر برای register/login/refresh فعال شد.
- Output Cache کوتاه‌مدت برای کاتالوگ و CMS عمومی اضافه شد.
- Response Compression برای API و Blazor فعال شد.
- Headerهای امنیتی پایه شامل CSP، X-Frame-Options، X-Content-Type-Options، Referrer-Policy و Permissions-Policy اضافه شد.
- Blazor از Antiforgery موجود استفاده می‌کند و API به خاطر Bearer Token در برابر CSRF مبتنی بر cookie متکی نیست.
- دسترسی به داده‌ها با EF Core و پارامترسازی queryها انجام می‌شود و Razor خروجی متن را encode می‌کند.
- Indexهای دیتابیس برای queryهای پرتکرار محصولات، سفارش‌ها، پرداخت‌ها، CMS، کاربران و توکن‌ها اضافه شد.
- لیست‌های سنگین عمومی pagination دارند و لیست‌های سنگین ادمین سقف محافظ دارند.
- SEO پایه تکمیل شد: meta description، OpenGraph، canonical URL، robots.txt و XML sitemap.
- بهینه‌سازی تصویر پایه با lazy loading، async decoding و fetch priority تصویر اصلی محصول انجام شد.
