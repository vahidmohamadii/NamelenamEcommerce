# NaderEcommerce

Clean Architecture starter for the multi-category e-commerce platform described in:

`C:\Users\Lenovo\Downloads\E-Commerce-Web-Application-Proposal.md`

## Current Phase

Phase 6 is complete. The project now includes the catalog, checkout and mock
payment flow, admin panel, CMS, security/performance/SEO hardening, seed data,
production configuration templates, deployment documentation, and automated
tests for auth, catalog readiness, admin product/order management, and the
order/payment workflow.

## Projects

- `NaderEcommerce.Domain`: domain entities and shared domain primitives.
- `NaderEcommerce.Application`: use cases, contracts, validation, and mapping.
- `NaderEcommerce.Infrastructure`: EF Core, SQL Server, and external integrations.
- `NaderEcommerce.WebApi`: ASP.NET Core REST API.
- `NaderEcommerce.BlazorWeb`: Blazor Web App frontend.
- `NaderEcommerce.BlazorWeb.Client`: client project generated by Blazor Auto interactivity.

## Local Commands

```powershell
dotnet restore
dotnet build
dotnet test
dotnet ef database update --project src/NaderEcommerce.Infrastructure/NaderEcommerce.Infrastructure.csproj --startup-project src/NaderEcommerce.WebApi/NaderEcommerce.WebApi.csproj
dotnet run --project src/NaderEcommerce.WebApi/NaderEcommerce.WebApi.csproj
```

Deployment guide: `docs/Deployment-Guide.fa.md`

API endpoints:

- Swagger: `https://localhost:7023/swagger`
- Health: `https://localhost:7023/health`
- Status: `https://localhost:7023/api/status`
- Register: `POST /api/auth/register`
- Login: `POST /api/auth/login`
- Refresh token: `POST /api/auth/refresh`
- Logout: `POST /api/auth/logout`
- Change password: `POST /api/auth/change-password`
- Current user: `GET /api/auth/me`
- Profile: `GET /api/account/profile`
- Update profile: `PUT /api/account/profile`
- Admin users: `GET /api/admin/users`
- Assign role: `POST /api/admin/users/roles`
- Activate/deactivate user: `PATCH /api/admin/users/{userId}/active`
